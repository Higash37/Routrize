# PRD - Product Requirements Document

## プロダクト概要

**Routrize** は、塾講師が「参考書ルート（カリキュラム）」を高速に作成し、保存・PDF出力・生徒配布・進捗ログ収集までできるB2B SaaS。

### 勝ち筋

「開いた瞬間にルートを作れる」→「ログインで保存と配布が解放」→「校舎運用が回る」

---

## MVPスコープ

### Must（実現すること）

#### A. ゲスト（ログイン不要）

- ルート作成（参考書を追加・期間設定・ガント表示）
- A4 PDFプレビュー/出力（`TRIAL` 透かしあり）
- 「保存」押下でログイン誘導 → ログイン後にクラウド保存

#### B. ログイン後（講師）

- ルートのクラウド保存/複製
- A4/A3 PDF出力（透かしなし）
- 生徒の作成（個人情報なし：店舗コード＋生徒ID＋PIN）
- ルートを生徒へ割当
- 生徒の学習ログ（日次：時間/教材/理解度）を収集
- 講師ダッシュボード（未入力・今週学習時間集計）

#### C. 組織

- 組織 → 店舗（校舎） → 講師の所属
- 店舗内共有（講師が作ったルートを校舎で見られる）

### Non-goals（やらないこと）

- AI推奨/自動ルート生成
- チャット、プッシュ通知、メール通知
- SNS共有、マーケット（出版社連携）
- 高度な相関分析/因果推定
- ネイティブアプリ（PWAでOK）

---

## 体験設計（最重要導線）

### 初回アクセス（ゲスト）

**URL**: `/`

- Route Builder（ルート編集画面）が即表示
- ヘッダー右に `ログイン` と `保存` と `PDF`
- ゲストで `保存` → ログインモーダル → ログイン成功後に routes 保存

### ログイン後（講師）

**URL**: `/app`

- 左ナビ: Dashboard / Routes / Students / Insights（MVPは軽い集計だけ）
- ルート編集は `/app/routes/:routeId`

### 生徒（別入口）

**URL**: `/s`

- 店舗コード + 生徒ID + PIN でログイン
- 今日の入力（10秒）: 学習時間（分）、教材（割当ルートから選択）、理解度、任意メモ
- 自分のログ履歴（リスト）

---

## ターゲットユーザー

| ロール | 行動 |
|---|---|
| 塾講師 / 校舎長 | 指導計画を作り、運用し、改善する |
| 生徒 | その日やった学習を10秒で記録する |
| 保護者（間接） | 何を、どのくらい、どう運用しているかを納得できる |
